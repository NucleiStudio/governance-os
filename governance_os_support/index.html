This macro makes it easy to prepare a mock testing runtime. It also avoid the developer from duplicating mocking code. However, the developer is still expected to write the correct imports for now. The mock runtime is composed of the system and bylaws pallets as all the other pallets of the Governance OS project are supposed to depend on those; so better not write the same code twice.
